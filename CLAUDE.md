# EnglishApp — Claude向け改善仕様書

## このアプリについて

小学校英語学習用のWebアプリ。React + TypeScript + Vite で構築。
Supabaseをデータベースとして使用。GitHub Pagesにデプロイ済み。

開発者：山田ゆうじ（プログラミング初心者・非エンジニア）
- 専門用語をできるだけ避けて説明すること
- 変更前に「何をするか・なぜするか・安全かどうか」を説明すること
- 大きな変更は一度に全部やらず、段階的に確認しながら進めること

## ポイント設計ルール（新モードを追加するときは必ず準拠）

チート（＝まっとうに学ばずに点を稼ぐ）を防ぐため、加点は必ず次を守る：

1. **加点は「まっとうな学習の成果」にひもづける**。単に「押した／完了した」だけでは点を出さない。
   - クイズ系は**正答率でスケール**し、半分未満なら加点なし（例：`addPoints(key, { multiplier: correctRatio })`、`ratio < 0.5` は加点しない）。
   - 選択式で「間違えると答えが浮き出る」ヒントがある場合、**一発正解だけを加点対象にカウント**（ヒントで正解しても加点は減る）。
   - 発音・音読系は合格ライン（accuracyScore）を満たしたときだけ。
2. **繰り返すほど加点は逓減し、最後は0**。これは `usePoints.addPoints` が全モード共通で処理する
   （`20→10→5→2→1→0`）。新モードも必ず `addPoints(activityKey, ...)` 経由にすること。
   同じ活動の連打で無限に稼げないのが前提。
3. **失敗（不合格）は clearCounts を増やさない**（＝逓減カウンタを消費しない）。合格したクリアだけ数える。
