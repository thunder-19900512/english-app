# 実装計画：ごほうびショップ（称号・きせかえ・クラスの木・背景画像）＋ローマ字撤去

> 2026-07-21 Fable作成。実装はOpusが本計画に沿って行う。
> 方針：**学習の注意を奪わない低刺激のごほうび**。たまごっち的な世話・育成ループは作らない。
> 「木」も見るだけ（世話なし）。ショップは買って終わり。

---

## 0. 前提：ポイントを絶対に減らさない設計（最重要）

現行の消失対策は「pointsは累計＝単調増加、同期はMath.maxマージ」で成立している。
**pointsから引き算してはいけない**。買い物は次の構造で実現する：

- `points` … 従来通り**累計**（減らない。maxマージ維持）
- 新規 `shop` jsonbカラムに使用額を持つ：
  ```
  shop: {
    spent: number,          // ショップでの累計消費（単調増加）
    donated: number,        // クラスの木への累計寄付（単調増加）
    owned: string[],        // 買った称号/テーマのid（和集合マージ）
    equippedTitle: string|null,   // 装備中の称号id
    equippedTheme: string|null,   // 装備中のテーマid
    bgImage: string|null,         // 背景画像のURL（Supabase Storage）
  }
  ```
- **残高 = points − spent − donated**（表示・購入判定に使う）
- 同期マージ規則（sync.tsのpush/pull両方に実装）：
  - `spent`・`donated` → **Math.max**（単調・消失事故に強い）
  - `owned` → **和集合**
  - `equippedTitle`・`equippedTheme`・`bgImage`（好みの設定＝単調でない）→
    **ローカルにshopオブジェクトが存在する場合のみローカル優先、無ければDB値**。
    （localStorageキー `shop_${id}` が丸ごと無い＝その端末は何も知らない、の意）
  - 安全モード（DB読込失敗時）：shopが空オブジェクトなら**送らない**
- localStorage: `shop_${studentId}` にJSONで保持

## 4. フック `src/hooks/useShop.ts`（新規）

- `shop` state（localStorage `shop_${id}` 起点、`shopUpdated` イベントで再読込）
- `balance = totalPoints − spent − donated`（usePointsのtotalPointsを利用）
- `buy(item)`: 残高チェック → spent += price, owned.push(id) → localStorage保存 → pushToSupabase → トースト`🎉「◯◯」を手に入れた！`
- `equipTitle(id|null)` / `equipTheme(id|null)`: 保存＋push
- `donate(amount)`: 残高チェック → donated += amount → 保存＋push
- `setBgImage(url|null)`: 保存＋push
- **加点は一切しない**（ポイント設計ルール準拠：ショップは出口のみ）

## 5. sync.ts 変更

- push: DB行の `shop` と上記マージ規則で統合して upsert に `shop` を追加
- pull: 同じマージで localStorage `shop_${id}` に書き戻し
- 安全モード分岐にも shop を追加（空なら送らない）

## 8. クラスの木 `src/components/tree/ClassTree.tsx`（新規、ルート `/tree`）

## 12. リスク・留意

- **背景画像は子どものアップロード**：他人には見えない設計なので低リスクだが、
  スタッフ画面から任意生徒のbgImageをクリアできる小ボタンを（生徒ごとビューに）付けておくと安心（任意・軽い）
- Storageのanon書き込みは既存アプリの信頼モデル（anonキーで全部）と同等。バケットはbackgrounds限定
- 木の寄付は「グループ対抗で煽りすぎない」トーンの文言に（「みんなで育てよう」主体、差分表示は小さく）
- 称号・きせかえの二重購入防止：owned済みは買えない
